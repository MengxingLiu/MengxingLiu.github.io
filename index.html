Computing speed contrast between reading from matfile and binary file

Veröffentlicht am 2015-09-08

In Neuroscience, we often deal with large data storage and management. Single fmri dataset or eeg dataset is usually 100+M. In matlab, reading a dataset into a variable in workspace means it is stored in memory and ready for computing. Well, the problem is, if we read a single large dataset into a variable, it could slow down the computing speed, obviously.
The good news is Matlab offers functions allowing us to read part of the single dataset into memory, instead of reading the whole file: matfile reads from mat file, and fread read from binary file.
The advantages are obviously.

  • You can save variables as big as you want into a single file, without worrying about read the file later.
  • You can access part of one variable if you want.
  • You can write/change variable in a large file without loading it, if you enable the permission.

Ok, if you are convinced, you may wondering which function you want, matfile or fread.
Recently, I’m tring to do some caculation, and I wanted to know which one is more efficient. So I run a time test. And It turn out that reading from binary file using fread is faster.

<< calcor2(1)
the calculating from mat finished!
Elapsed time is 176.910272 seconds.
<< calcor2(2)
the calculating from bin finished!
Elapsed time is 34.537742 seconds.

Testing function(if you want to do some test, just write a simple loop :) ):

function calcor(method,result_path)
%% this function offers two method to read the map of orignal data and calc the correlation between every possible pair.
tic;
    if nargin == 0
        method = 1;    
    end
    result_path = 'I:\StoryISC\listen_chn_result\data_mat\';
    if method == 1

        %% method one
        % read map data from mat files

        name_list = dir(fullfile(result_path,'*.mat'));
        name_list = strcat(result_path,extractfield(name_list(1:4),'name'));
        bmask = load_nii('I:\StoryISC\2group\ts_all\mask_biling+tlrc.nii');
        bmask = logical(single(bmask.img));
        tmp = matfile(name_list{1,1});
        [x,y,z,t] = size(tmp.I);
        cData = zeros(y,z,t,length(length(name_list)));
        INDS = find(triu(ones(length(name_list),length(name_list)),1));
        corMatData = zeros(y,z,length(INDS));
        cormat = zeros(x,y,z,length(INDS));
        % do calc correlation
        for vx = 1:x

            for k = 1:length(name_list)
                tmp = matfile(name_list{1,k});
                cData(:,:,:,k) = tmp.I(vx,:,:,:);
            end
            for vy = 1:y
                for vz = 1:z
                    if bmask(vx,vy,vz)
                        ts = squeeze(cData(vy,vz,:,:));
                        xc = ts - repmat(sum(ts)/t,t,1);
                        c1 = (xc' * xc) / (t-1);
                        d1 = sqrt(diag(c1));
                        dd = d1*d1';
                        dd(1:k+1:end) = diag(c1); % remove roundoff on diag
                        r1 = c1 ./ dd;
                        r1 = r1(INDS);
                        % save all correlations:
                        corMatData(vy,vz,:) = r1;

                    end
                end
            end
            cormat(vx,:,:,:) = corMatData;
            % disp(['the ',num2str(vx),'th',' finished!'])

        end
        save([result_path 'cormat.mat'],'cormat');
        disp(['the ','calculating from mat ',' finished!'])
    else
        %% method two
        name_list = dir(fullfile(result_path,'*.bin'));
        name_list = strcat(result_path,extractfield(name_list(1:4),'name'));
        bmask = load_nii('I:\StoryISC\2group\ts_all\mask_biling+tlrc.nii');
        bmask = logical(single(bmask.img));    
        x = 54;y = 64; z = 50; t = 266;
        INDS = find(triu(ones(length(name_list),length(name_list)),1));
        corMatData = zeros(y,z,length(INDS));
        cormat = zeros(x,y,z,length(INDS));
        cData = zeros(t,y,z,length(length(name_list)));
        for vx = 1:x

            for k = 1:length(name_list)
                tmp = memmapfile(name_list{1,k},'format',{'single',[t,y,z],'tyz'});
                cData(:,:,:,k) = tmp.Data(vx).tyz;
            end
            for vy = 1:y
                for vz = 1:z
                    if bmask(vx,vy,vz)
                        ts = squeeze(cData(:,vy,vz,:));
                        xc = ts - repmat(sum(ts)/t,t,1);
                        c1 = (xc' * xc) / (t-1);
                        d1 = sqrt(diag(c1));
                        dd = d1*d1';
                        dd(1:k+1:end) = diag(c1); % remove roundoff on diag
                        r1 = c1 ./ dd;
                        r1 = r1(INDS);
                        % save all correlations:
                        corMatData(vy,vz,:) = r1;
                    end
                end
            end
            cormat(vx,:,:,:) = corMatData;
            % disp(['the ',num2str(vx),'th',' finished!'])

        end
        disp(['the ','calculating from bin ',' finished!'])

    end
toc;
end

Inter-subject correlation(ISC) in AFNI

Veröffentlicht am 2015-05-28

Nothing Special.
Below are commands performing Inter-subject correlation(ISC) based analysis of fMRI data using AFNI.
I assume you already know something about Shell and AFNI when you are seeing this.
As you may know, there are at least two ways to evaluate the coefficient between subjects in a group,which result in nearly identical correlation maps(Honey,2012)
Here only one of them are introduced.
An correlation coefficients set is created through map of each subject correlating with the average map of the other individuals.
Be sure you get rid of the one time course which you are going to do correlation between it and group when you are averaging the time course of group.

set datapath = /directory where your datatsets are
cd "$anapath"
set listening = (listening_chn_sub01.nii listening_chn_sub02.nii listening_chn_sub03.nii \
listening_chn_sub04.nii listening_chn_sub05.nii listening_chn_sub06.nii \
listening_chn_sub07.nii listening_chn_sub08.nii listening_chn_sub09.nii \
listening_chn_sub10.nii listening_chn_sub11.nii listening_chn_sub12.nii \
listening_chn_sub13.nii listening_chn_sub14.nii listening_chn_sub15.nii listening_chn_sub16.nii) # all of your time course in one group
set subject = (sub01 sub02 sub03 sub04 sub05 sub06 sub07 sub08 sub09 sub10 sub11 sub12 \
sub13 sub14 sub15 sub16)
set i = 1
while ( $i <= 16 )
set sub=(`echo ${subject} | awk '{print $'$i'}'`)
set listening=(`echo $listening | awk '{$'$i'=""; sub(" ", " "); print}'`)
set reading=(`echo $reading | awk '{$'$i'=""; sub(" ", " "); print}'`)
3dMean -prefix mean_chn_listening.nii $listening
3dTcorrelate -pearson -polort -1 -prefix r_listening_chn_"$sub".nii listening_chn_"$sub".nii mean_chn_listening.nii
3dcalc -a r_listening_chn_"$sub".nii -expr '0.5*(log((1+a)/(1-a)))' -prefix z_listening_chn_"$sub".nii # transform the r value to z
rm mean_chn_listening.nii # be sure you remove the average time course map when you are done one loop
@ i ++
end
# Now that all of the correlation maps are created for each subject. You can do a one sample t-test against 0 to see if there are any voxels showing significant correlation in this group.
3dttest++ -prefix ttest_listening_chn.nii \
-setA z_listening_chn_*.nii

Crawling Weekly Address on Whitehouse.org using Python

Veröffentlicht am 2015-01-08

When I was preparing for the IELTS, I tried to download the MP3 of Weekly Address on Whitehouse.org. But there were hundreds of files from 2012-2016. As a man who prosess Laziness, Impatience and Hubris ( three virtues of software design accordding to Larry Wall, what a coincidence! ), I could not tolerate this repetitive work, which should be the job of computer. Then I thought, hey, Python may be a good idea. So I spent hours on the book and internet finding a solution to make the computer do its work using python. I have to say, it’s really a good way to kill time.
Below is the codes.

import urllib.request
import re
import sys

import time


def read_html(per_url):
    __html__ = urllib.request.urlopen(per_url).read()
    html_str = __html__.decode('utf-8')
    return html_str


def dl_mp4(per_url):
    __html__ = read_html(per_url)
    pattern = re.compile(r'https.{,90}\.mp3')
    name_pattern = re.compile(r'\d{6}_.{,16}\.mp3')
    vurl = pattern.findall(__html__)
    name = name_pattern.findall(__html__)
    if vurl and name:    
        urllib.request.urlretrieve(vurl[0],"%s" % name[0])
        print(vurl)
        print("%s" % name[0])    
    else:
        pass


print("Start Time:",time.strftime('%H:%M:%S'))
page_number = 0
urls = []
while page_number <= 1:
    html = read_html('https://www.whitehouse.gov/briefing-room/weekly-address?page=%d' % page_number)
    links = re.findall(r'the-press-office.*weekly-address.*"', html)
    url_per_page = ["https://www.whitehouse.gov/"+x[:-1] for x in links[1:]]
    page_number += 1
    urls += url_per_page
    print("Request Time:",time.strftime('%H:%M:%S'))
print(urls)

n = 0
for url in urls:
    dl_mp3(url)
    n += 1
    print("Receive %d file:" % n ,time.strftime('%H:%M:%S'))

print("Finish Time:",time.strftime('%H:%M:%S'))
